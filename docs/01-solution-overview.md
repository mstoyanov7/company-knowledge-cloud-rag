# Solution Overview

## Problem

Company onboarding knowledge is often split across:

- OneNote notebooks
- SharePoint sites and document libraries
- scattered onboarding files, PDFs, Word documents, presentations, and internal notes

Newcomers waste time searching manually, and answers become outdated when content changes.

## Proposed solution

Build a cloud-based RAG system that sits behind Open WebUI.

The user interacts with Open WebUI.  
Open WebUI sends the question to a custom backend.  
The backend retrieves relevant company knowledge from a pre-built and continuously updated index.  
The answer is then generated by any compatible LLM selected by the user or administrator.

## Main requirements

### Functional

- connect to SharePoint through Microsoft Graph
- connect to OneNote through Microsoft Graph
- fetch and normalize content
- index content in a cloud vector database
- answer questions with citations
- support multiple LLM providers
- refresh the index incrementally

### Non-functional

- low retrieval latency
- scalable for many users
- secure by default
- explainable via source citations
- resilient to API failures and throttling
- maintainable as a company service

## Boundary of responsibility

### Open WebUI should do

- user authentication if integrated that way
- chat UI
- chat history
- model selection
- sending prompts to your backend or model endpoint

### Your backend should do

- source sync
- parsing and chunking
- embedding
- indexing
- metadata and ACL handling
- retrieval
- reranking
- prompt building
- citation generation
- freshness handling

## Recommended architectural style

Use event-driven plus scheduled synchronization:

- event-driven where the source supports it well
- scheduled polling where event-driven sync is not enough
- nightly reconciliation to repair missed events
