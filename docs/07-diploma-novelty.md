# Diploma Novelty Direction

## Weak novelty statement

"An AI chatbot for company onboarding."

This is too generic.

## Stronger novelty statement

"A model-agnostic enterprise Cloud-RAG architecture for onboarding assistance that combines incremental synchronization from SharePoint and OneNote, ACL-aware retrieval, and freshness-aware citation generation through a unified Open WebUI interface."

## Practical research focus options

### Option A - Freshness-aware RAG
Compare answer quality and staleness between:

- static indexing
- periodic polling only
- incremental event + polling strategy

### Option B - ACL-aware enterprise retrieval
Study how chunk-level permission filtering affects retrieval quality and safety.

### Option C - Adaptive source sync
Design a scheduling policy that changes polling frequency based on source activity.

### Option D - Citation trust in onboarding systems
Measure whether source-backed answers improve user trust and task completion.

## Best balanced choice

For a diploma with both engineering and research value:

- freshness-aware indexing
- ACL-aware retrieval
- source-cited answer generation

This gives you architecture, implementation, and measurable evaluation.
