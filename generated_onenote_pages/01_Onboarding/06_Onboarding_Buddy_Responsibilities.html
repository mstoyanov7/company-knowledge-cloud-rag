Onboarding Buddy Responsibilities

What is expected from an assigned onboarding buddy.

Overview

Onboarding Buddy Responsibilities is maintained by People Operations for new hires, managers, onboarding buddies. This onboarding guide what is expected from an assigned onboarding buddy.

Ownership

Owner teamPeople Operations
Audiencenew hires, managers, onboarding buddies
SystemsSlack, GitHub, HR Portal

Key Facts

  • The buddy is a peer engineer, not the new hire's manager.
  • Buddies commit roughly 3 hours per week for the first two weeks.
  • The buddy reviews the new hire's first pull request within the same day.
  • Buddy assignments rotate so no engineer buddies more than once per quarter.

Onboarding Steps

  1. Introduce the new hire to the team and key contacts.
  2. Pair on the first starter ticket.
  3. Review the first pull request promptly.

What To Keep

Keep the completion checklist, manager confirmation, buddy sign-off with the related ticket or record.
