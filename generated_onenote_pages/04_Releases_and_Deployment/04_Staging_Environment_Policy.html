Staging Environment Policy

What staging is for and how it stays trustworthy.

Context

Staging Environment Policy is the release process maintained by Platform Team for release managers, on-call engineers, team leads. It covers what staging is for and how it stays trustworthy. Staging mirrors production configuration and is refreshed with anonymized data weekly.

Owner and Systems

Deployment Process

  1. Deploy your change to staging via the pipeline.
  2. Run the relevant smoke and integration checks.
  3. Leave staging in a clean state for the next team.

Key Facts

  • Staging mirrors production configuration and is refreshed with anonymized data weekly.
  • Staging is shared; do not run destructive load tests without booking a window.
  • A change must pass staging before it is eligible for production.
  • Staging uses real third-party sandboxes, not mocks.
