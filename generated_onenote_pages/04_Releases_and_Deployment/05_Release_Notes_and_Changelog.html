Release Notes and Changelog

How customer-facing release notes are produced.

Overview

Release Notes and Changelog is the release process maintained by Product Engineering for release managers, on-call engineers, team leads. It covers how customer-facing release notes are produced. Customer release notes are generated from PR titles labeled 'changelog'.

Ownership

Owner teamProduct Engineering
Audiencerelease managers, on-call engineers, team leads
SystemsGitHub, Product Portal

Key Facts

  • Customer release notes are generated from PR titles labeled 'changelog'.
  • Notes are published every second Wednesday with the sprint release.
  • Breaking changes require a migration note and 30 days notice to customers.
  • Internal-only changes are excluded from the public changelog.

Deployment Process

  1. Label customer-facing PRs with 'changelog'.
  2. Review the generated draft before publishing.
  3. Add migration notes for breaking changes.

What To Keep

Keep the release ticket, approval record, rollback owner, monitoring link with the related ticket or record.
