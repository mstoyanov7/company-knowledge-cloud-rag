Rollback Procedure

How to roll back a bad production release quickly.

Summary

Rollback Procedure is maintained by Release Management for release managers, on-call engineers, team leads. This release process how to roll back a bad production release quickly.

Key Facts

  • Rollback is a one-command redeploy of the previous known-good image tag.
  • The rollback target is always the image tagged last-stable.
  • Database migrations are backward compatible for at least one release to make rollback safe.
  • If a migration is not reversible, a forward-fix is used instead of rollback.

Deployment Process

  1. Declare a rollback in the incident channel.
  2. Run the rollback command to last-stable.
  3. Confirm error rate returns to baseline.
  4. Open a follow-up to forward-fix the root cause.

Commands

Run from a clean shell and keep the output with the setup or incident record.

nw deploy rollback --service <service> --to last-stable
nw deploy status --service <service>

What To Keep

Keep the release ticket, approval record, rollback owner, monitoring link with the related ticket or record.
