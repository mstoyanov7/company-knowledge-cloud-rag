Code Review Standards

What reviewers check and the expected turnaround.

Purpose

Code Review Standards is maintained by Developer Platform for all software engineers. This engineering standard what reviewers check and the expected turnaround.

Key Facts

  • Reviewers respond within one business day; if unavailable, reassign.
  • Reviews focus on correctness, tests, readability, and security, in that order.
  • A pull request over 400 changed lines should be split unless it is generated code.
  • Authors must respond to every review comment before merge.
  • Approving a PR means you share responsibility for the change.

How We Apply It

  1. Read the ticket and PR description before the diff.
  2. Check tests exist and cover the change.
  3. Leave actionable comments, not vague ones.
  4. Approve only when you would be comfortable owning the change.
