Testing Strategy

The testing pyramid and coverage expectations.

Summary

Testing Strategy is the engineering standard maintained by Quality Engineering for all software engineers. It covers the testing pyramid and coverage expectations. Target coverage is 80% lines for changed code, enforced on the diff, not the whole repo.

Key Facts

  • Target coverage is 80% lines for changed code, enforced on the diff, not the whole repo.
  • Unit tests run on every PR; integration tests run on merge to main.
  • End-to-end tests run nightly against staging.
  • Flaky tests are quarantined within 24 hours and fixed within a week.

How We Apply It

  1. Write unit tests alongside the code.
  2. Add an integration test for any new external dependency.
  3. Keep tests deterministic and independent.
