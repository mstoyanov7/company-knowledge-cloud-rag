Architecture Decision Records

How and when to write an ADR.

Overview

Architecture Decision Records is maintained by Architecture Office for all software engineers. This engineering standard how and when to write an ADR.

Ownership

Owner teamArchitecture Office
Audienceall software engineers
SystemsGitHub, Architecture Wiki

Key Facts

  • ADRs live in docs/adr/ in each repository and are numbered sequentially.
  • Write an ADR for any decision that is costly to reverse, such as a database or framework choice.
  • An ADR has four parts: context, decision, status, and consequences.
  • ADRs are immutable once accepted; a new ADR supersedes an old one.

How We Apply It

  1. Copy the ADR template into docs/adr/.
  2. Fill in context, decision, and consequences.
  3. Open a PR and get architecture review.
  4. Mark the ADR accepted when merged.
