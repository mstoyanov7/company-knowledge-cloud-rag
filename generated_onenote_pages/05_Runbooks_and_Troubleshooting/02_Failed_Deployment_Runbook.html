Failed Deployment Runbook

What to do when a production deployment fails health checks.

Purpose

Failed Deployment Runbook is maintained by Release Management for on-call engineers, support engineers. This runbook what to do when a production deployment fails health checks.

Key Facts

  • The pipeline automatically halts rollout if readiness checks fail for 3 minutes.
  • A failed deploy never takes down the running version; old pods keep serving.
  • Most failures are missing config or a failed migration.
  • If unsure, roll back to last-stable rather than debugging in production.

Investigation Steps

  1. Read the pipeline logs for the failing step.
  2. Check for missing config or migration errors.
  3. Roll back to last-stable if the fix is not obvious.
  4. Open an incident if customers were affected.
