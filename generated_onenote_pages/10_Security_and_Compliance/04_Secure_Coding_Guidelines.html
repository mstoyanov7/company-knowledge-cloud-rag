Secure Coding Guidelines

Baseline practices to avoid common vulnerabilities.

Context

Secure Coding Guidelines is maintained by Security Engineering for all employees, engineers, auditors. This security policy baseline practices to avoid common vulnerabilities.

Control Process

  1. Use parameterized queries and safe APIs.
  2. Validate and encode untrusted input.
  3. Resolve high-severity dependency alerts before merge.

Key Facts

  • Never build SQL with string concatenation; use parameterized queries.
  • All input crossing a trust boundary is validated and encoded on output.
  • Dependencies are scanned on every PR and high-severity findings block merge.
  • Secrets are read from the vault, never hardcoded or committed.
