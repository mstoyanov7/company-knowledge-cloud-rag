Data Classification Policy

The four data classes and how to handle each.

Overview

Data Classification Policy is the security policy maintained by Security Engineering for all employees, engineers, auditors. It covers the four data classes and how to handle each. Data is classified as Public, Internal, Confidential, or Restricted.

Ownership

Owner teamSecurity Engineering
Audienceall employees, engineers, auditors
SystemsSecurity Portal, Data Catalog

Key Facts

  • Data is classified as Public, Internal, Confidential, or Restricted.
  • Customer personal data is always at least Confidential.
  • Restricted data, such as payment card data, may never be stored in logs or screenshots.
  • Confidential data must be encrypted at rest and in transit.

Control Process

  1. Determine the highest class of data you handle.
  2. Apply the matching storage and sharing rules.
  3. Ask Security if you are unsure.
