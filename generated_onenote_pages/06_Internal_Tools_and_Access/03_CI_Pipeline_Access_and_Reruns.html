CI Pipeline Access and Reruns

How to view, rerun, and debug CI jobs.

Overview

CI Pipeline Access and Reruns is the tool access guide maintained by Build Infrastructure for engineers and team leads. It covers how to view, rerun, and debug CI jobs. CI runs on GitHub Actions with self-hosted runners labeled nw-linux and nw-macos.

Ownership

Owner teamBuild Infrastructure
Audienceengineers and team leads
SystemsGitHub, CI Pipeline, Service Desk

Key Facts

  • CI runs on GitHub Actions with self-hosted runners labeled nw-linux and nw-macos.
  • Anyone on the repo can rerun a failed job; only owners can edit workflows.
  • A job stuck in queue for over 10 minutes usually means no runner is available.
  • Build artifacts are retained for 14 days.

Request Flow

  1. Open the Actions tab on the repository.
  2. Rerun the failed job or download logs.
  3. Escalate to Build Infrastructure if no runner picks it up.

What To Keep

Keep the access ticket, owner approval, role mapping with the related ticket or record.
