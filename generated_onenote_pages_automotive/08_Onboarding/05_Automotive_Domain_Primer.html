Automotive Domain Primer

Introduces common automotive software concepts used by engineering teams.

Buddy Note

Automotive Domain Primer is meant to be used by a new hire and a buddy sitting together, not by someone silently checking boxes. This page introduces common automotive software concepts used by engineering teams.

The buddy should explain where this fits in daily engineering work and should leave enough room for questions. A person can finish the form and still be blocked if they do not know which system to open first on Monday morning.

First Week Context

Start with domain glossary, then make sure architecture map and signal examples are not only assigned but usable. If a tool asks for extra approval, open the service desk ticket while the new hire is still in the onboarding session.

The page should be updated when the team changes tools, review habits, safety rules, or release cadence. Old onboarding pages create slow first weeks because people follow steps that no longer match the team.

What Should Be Ready

  • Domain glossary maintained by Systems Engineering.
  • Architecture map available before work starts.
  • Signal examples linked from the ticket, request, or workspace.
  • Manager, buddy, start date, team assignment, and required training dates.

How To Know It Worked

The useful signal is not only that the page was read. The new hire should have enough access to use Requirements Traceability, know where Architecture Wiki fits, and understand when to ask about Vehicle Log Portal.

Completion

  • The new hire has the required access, equipment, and owner contacts.
  • Onboarding evidence includes domain glossary, architecture map, signal examples.
  • The related systems are available: Requirements Traceability, Architecture Wiki, Vehicle Log Portal.
  • The manager or buddy confirms the onboarding step is complete.

Onboarding Notes

  • Missing first-week access slows down training and code review participation.
  • Safety and security training should be completed before bench or production access is granted.
  • Managers should update the page when team-specific onboarding changes.
