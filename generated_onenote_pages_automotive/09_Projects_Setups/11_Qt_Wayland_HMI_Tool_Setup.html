Qt Wayland HMI Tool Setup

Sets up the Qt Wayland HMI toolchain for cockpit prototypes and target display validation.

What This Workspace Is For

Qt Wayland HMI Tool Setup is used when an engineer needs a working local copy of the project, not just read access to the repository. This page sets up the Qt Wayland HMI toolchain for cockpit prototypes and target display validation.

The setup normally starts with access to Qt Creator, then moves through the required artifacts: Qt project, Wayland profile, display config. The important part is to keep the machine, the project configuration, and the verification output aligned. If those three do not match, the project can appear to work locally and still fail during integration.

HMI Platform keeps this page current because small toolchain changes matter in automotive work. A different compiler, DBC file, ODX package, schema version, or bench profile can change the result enough to waste a full validation session.

For Windows laptops, use the PowerShell section first and keep long paths enabled in Git. For Linux laptops or WSL, use the shell section. If the project touches lab equipment, check the reservation or bench status before running anything that talks to hardware.

Qt Toolchain

Use this path on Linux, WSL, or a standard validation workstation.

mkdir -p ~/automotive/qt-wayland-hmi-tool
cd ~/automotive/qt-wayland-hmi-tool
git clone ssh://git.company.local/hmi/qt-wayland-hmi-tool.git .
cmake -S . -B build -DCMAKE_PREFIX_PATH="$QT_HOME" -DENABLE_WAYLAND=ON
cmake --build build --target hmi_shell --parallel
WAYLAND_DISPLAY=wayland-1 ./build/bin/hmi_shell --profile cockpit-bench

Windows Setup

Use this path on a company Windows laptop or before switching into WSL.

$workspace = Join-Path $env:USERPROFILE "automotive\qt-wayland-hmi-tool-setup"
New-Item -ItemType Directory -Force $workspace | Out-Null
Set-Location $workspace
git config --global core.longpaths true
winget install -e --id Git.Git
winget install -e --id Kitware.CMake
winget install -e --id Ninja-build.Ninja
winget install -e --id WiresharkFoundation.Wireshark
git clone ssh://git.company.local/hmi/qt-wayland-hmi-tool.git .
Write-Output "Run the Linux or project-specific verification block after tools are installed."

How Engineers Usually Work With It

Engineers usually start this workspace during onboarding, release preparation, or a supplier handoff. The first run should be treated like a controlled setup: download the project material, install the exact tools, run the verification command, and then save the output.

Once the project is running, daily work should happen on a feature branch with the same local profile used during setup. If display config changes, repeat the verification command before blaming the application code.
