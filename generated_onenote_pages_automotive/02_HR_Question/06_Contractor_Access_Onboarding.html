Contractor Access Onboarding

Defines onboarding requirements for contractors who need source code, test bench, or supplier portal access.

Policy Explanation

Contractor Access Onboarding should read like an answer from HR to an employee who wants to know what happens next. This page defines onboarding requirements for contractors who need source code, test bench, or supplier portal access.

The page should explain the rule in plain language first, then point to Identity Provider for the actual request. Managers can use it to understand timing and approval responsibility without copying private employee data into shared notes.

Employee Path

The employee normally starts with contractor profile and checks whether access request is needed. If the case depends on a date, contract type, country, or release freeze, the manager should ask People Operations before promising an exception.

When the request is complete, the final answer should be recorded in Identity Provider or the approved case system. The shared page stays general so future employees can reuse it.

Useful Inputs

  • Contractor profile maintained by People Operations.
  • Access request available before work starts.
  • End date linked from the ticket, request, or workspace.
  • Employee eligibility, request date, manager approval when required, and policy source.

What HR Checks

  • Owner team is People Operations and has accepted the request or review.
  • Required artifacts are available: contractor profile, access request, end date.
  • Relevant systems are checked: Identity Provider, Vendor Portal, Service Desk.
  • The final decision or completion note is attached to the ticket, request, or record.

Privacy Note

Keep employee-specific information in Identity Provider or the approved HR case tool. Notes should explain the rule and the expected path, not store private employee details.

Exceptions

Escalate unclear ownership, missing evidence, blocked approvals, or exceptions to People Operations. The usual requester is the employee or manager, and the standard service desk or program channel should hold the final note.
