Reference Note

Naming Convention Catalog is a shared reference for teams that need the same wording, decision, or convention. This page lists naming rules for repositories, branches, services, feature flags, test benches, and artifacts.

It should be short enough to stay readable but specific enough that two teams do not interpret the same term or rule differently. The owner should keep examples current because examples are usually what people copy.

Maintained By

Owner teamArchitecture Office
Audienceall technical employees
Review rhythmReview quarterly or when the owner team changes process.
Primary systemsArchitecture Wiki, GitHub Enterprise, Artifact Registry

How Teams Use It

Teams usually open this page while preparing a review, naming something new, writing a decision record, or checking an audit detail. The useful material is naming rule, prefix, example.

When the reference affects repositories, artifacts, or architecture records, update Architecture Wiki, GitHub Enterprise, and Artifact Registry in the same change window.

Useful Material

  • Naming rule maintained by Architecture Office.
  • Prefix available before work starts.
  • Example linked from the ticket, request, or workspace.
  • Current owner, review date, linked decision, and affected teams.

Review Habit

When this reference changes, link the decision or review note in Architecture Wiki and tell the teams that depend on GitHub Enterprise. If the change affects artifacts, also update Artifact Registry.

Checks

  • Owner team is Architecture Office and has accepted the request or review.
  • Required artifacts are available: naming rule, prefix, example.
  • Relevant systems are checked: Architecture Wiki, GitHub Enterprise, Artifact Registry.
  • The final decision or completion note is attached to the ticket, request, or record.
