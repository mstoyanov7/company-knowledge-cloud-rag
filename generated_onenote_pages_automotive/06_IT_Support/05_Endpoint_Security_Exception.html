Endpoint Security Exception

Explains how to request temporary endpoint security exceptions for compilers, lab tools, or hardware drivers.

Service Desk Note

Endpoint Security Exception is a service desk note for support cases that need a repeatable path. This page explains how to request temporary endpoint security exceptions for compilers, lab tools, or hardware drivers.

The support agent should identify the requester, affected asset or account, business reason, and urgency before changing anything. This is especially important when the request touches engineering tools, test benches, VPN, MFA, or production-adjacent systems.

Support Ownership

Owner teamSecurity Operations
Audienceemployees, service desk agents, endpoint engineers
Review rhythmReview when access rules, tooling, or ownership changes.
Primary systemsEndpoint Console, Security Review, Service Desk

Triage

Start with exception reason and verify it against Endpoint Console. Then check file hash and expiry date so the ticket has enough detail for another agent to continue the work if the first responder is unavailable.

Once the fix is applied, tell the requester exactly what changed and what they should try next. The ticket should not close until the requester confirms the result or the support window expires.

Ticket Closure Checks

  • Owner team is Security Operations and has accepted the request or review.
  • Required artifacts are available: exception reason, file hash, expiry date.
  • Relevant systems are checked: Endpoint Console, Security Review, Service Desk.
  • The final decision or completion note is attached to the ticket, request, or record.

Requester Communication

Tell the requester what changed, what still needs their action, and where the ticket is recorded. For this page, the normal systems are Endpoint Console, Security Review, Service Desk.

Support Risks

  • Identity verification is required before access, MFA, or device state changes.
  • Temporary exceptions need expiry dates and owners.
  • Tickets should not be closed until the requester confirms the fix.
